Purpose: Users on 4 GB-VRAM GPUs (currently falling back to CPU on the in-process Python path) get GPU-backed cloning via Q4_K_M; users on mid-range hardware get the recommended Q8_0 balance; high-VRAM users can keep BF16/F32. On any failure the existing `OmniVoiceBackend` remains the fallback so no user is left without a cloning engine.

Output: 7 GGUF-* requirements implemented + SPIKE-01 ADR ratified with the actual commit SHA the build was pinned to.
